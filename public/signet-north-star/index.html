This is not a spec. A spec says what to build. This note says why the thing is worth building, what kind of outcome it is supposed to protect, and what it should feel like when it is working.

Signet exists because context, care, craft, authority, and continuity are real things, and most software only preserves pieces of them. A person builds up years of judgment, habits, relationships, source material, corrections, preferences, projects, failures, taste, and hard-won world-models. Some tools preserve part of that record. Some agents can be warmed up with context. Some platforms offer exports, sync, memory, or history. But the continuity is usually fragmented, platform-scoped, hard to inspect, hard to move, and easy to compress into a summary that loses the living structure underneath.

Signet should repair that fragmentation.

It should make a user's world legible to agents without stealing it from the user. It should let agents act with context without turning context into mush. It should preserve the raw artifacts, expose the semantic shape, make authority inspectable, and keep behavior portable across tools, harnesses, machines, and models.

01

The artifact is ground truth

The graph is not God. The artifact is ground truth.

A transcript, an Obsidian note, an email, a file, a repo, a canvas, a source clipping, a generated asset, a user correction, a saved memory: these are not just inputs for a memory database. They are the record of what happened. They carry provenance, texture, uncertainty, and consequence.

The semantic layer exists because reading every artifact every time is too expensive. It is a navigation layer, not a replacement for reality. A derived claim, summary, memory, entity, or edge should always be able to lead back to the source that justifies it. If it cannot, it is gossip with a database behind it.

Signet should be source-native first. It should mount the user's real world — vaults, repos, emails, documents, transcripts, media, tasks, notes — and make that world agent-readable without pretending the derived graph is more real than the artifacts underneath.

02

The operational world model

Signet is not just personal memory. That frame is too small.

The stronger target is an agent-first operational ontology: a model of the world an agent has to act inside. Not a decorative knowledge graph. Not a starfield of semantic vibes. A working model of sources, objects, claims, policies, actions, permissions, people, projects, tools, and consequences.

Good memory is not storage. Good memory is not even recall. Great memory is consequence-aware context.

An agent should be able to ask:

  • what is this task about?
  • what source is authoritative?
  • what evidence supports this claim?
  • what changed since last time?
  • what actions are allowed?
  • what needs approval?
  • what should not be disclosed here?
  • what will this action affect?

That is the useful kind of world model. It does not need to predict the future in a grand AGI sense. It needs to model operational consequences well enough that an agent can act without trampling source truth, user intent, or permission boundaries.

03

The human reason

The last few days made the emotional stakes clearer.

Craft teaches what real competence feels like. Visual effects was not just a job or a hobby; it was a way of understanding systems in service of an outcome. A shot works or it does not. The image holds up or it betrays itself. That kind of craft builds taste, attention, and a bullshit detector.

A system that ignores craft does not merely lose trivia. It loses judgment. It loses the living process that made the output valuable.

That is why institutional craft amnesia matters. Disney and Marvel are not only a story about layoffs or AI. They are a warning about what happens when an institution archives the outputs of human judgment, then forgets the people, continuity, and process memory that made those outputs possible. AI can become a laundering layer for accumulated craft: management treats an archive of judgment like an asset library, then calls the living workers replaceable.

Signet should be built against that pattern.

It should preserve the connection between output and origin, action and authority, claim and evidence, agent behavior and the identity files that shaped it. It should make it harder for context to be extracted, flattened, and reused without memory of where it came from or what permissions governed it.

04

Authority is part of memory

Agents are going to act across more surfaces: ACP, UCP, MCP, A2A, AP2, wallets, payment rails, creative tools, codebases, browsers, calendars, inboxes, and whatever comes next.

The important question is not only whether an agent can do the action. The important question is who the agent is, why it has access, what it is allowed to do, what source context it is relying on, and what evidence proves it stayed inside its authority.

That means authority is not a sidecar. It is part of memory.

A delegated purchase, a code edit, a generated video, a calendar action, a published post, a graph mutation, or an identity-file patch all need some form of source-backed authority. Signet's natural role is to hold the portable identity, permissions, policies, evidence, and audit trail that travel underneath the protocol war.

05

Memory has different shapes

One of the most important corrections is that memory should not be forced into one substrate.

Different lessons belong in different forms:

  • graph facts belong in the ontology with provenance;
  • behavioral lessons belong in identity files like AGENTS.md or SOUL.md;
  • repeated procedures belong in skills;
  • source-backed concepts belong in literature notes;
  • permissions and delegation boundaries belong in authority artifacts;
  • experiment outcomes belong in evaluated run records;
  • raw events belong in source artifacts.

A single omnivorous memory database is not the goal. The goal is routing. The system should learn what kind of thing a lesson is, then put it where it can actually change future behavior without losing evidence.

That is why identity files matter. A graph can preserve provenance and connective tissue, but an identity file directly shapes the next agent turn. A scheduled dream session that reviews transcripts, asks what the agent did well or poorly, and proposes a small patch to AGENTS.md may deliver more practical self-improvement than a much fancier ontology mutation.

06

Dreaming is the maintenance loop

Signet should not silently mutate itself on the hot path.

The hot path is where an agent does the task. The dreaming path is where the system reviews what happened, analyzes outcomes, and proposes durable changes. That pass needs evidence, diffs, review, rollback, and metrics. It should not produce vibe summaries. It should produce mutation proposals.

ASI-Evolve gives the operating pattern:

  1. learn from prior cognition and source artifacts;
  2. design a candidate change;
  3. experiment or evaluate against real traces;
  4. analyze the result into a reusable lesson;
  5. route that lesson into the right memory shape.

The analyzer is the key. Without analysis, the system has logs and scores. With analysis, expensive traces become cognition. A failed agent session can become an identity-file patch. A repeated workflow can become a skill. A stale fact can become a graph repair. A bad retrieval can become a retrieval-contract change. A risky action can become an authority boundary.

Dreaming is how experience becomes better future behavior.

07

The product promise

Signet should feel like continuity that follows you.

It should make the user's context portable across harnesses instead of trapping it inside Claude, Codex, Cursor, OpenClaw, Hermes, or whatever comes next. It should make setup boring. It should make source connection easy. It should make memory inspectable. It should make identity and skills durable. It should make agent action safer because provenance and authority are always nearby.

The daemon should be boring substrate: source truth, APIs, graph state, proposal application, transcripts, artifacts, and durable storage. Agents and harnesses can provide the intelligence. Cron jobs, scheduled dream passes, CLI tools, and review surfaces can do the maintenance. The system should be open, inspectable, and harness-neutral without becoming ceremony-heavy.

Low-touch usage still matters. An agent should be able to recall, remember, ask for context, inspect evidence, and list allowed actions without navigating a bureaucratic ontology. The richness should sit underneath. The everyday interface should stay humane.

08

The North Star

Signet is a source-native, agent-readable continuity layer for people and organizations.

It preserves artifacts as ground truth, builds semantic shortcuts with provenance, routes lessons into the right memory shapes, and gives agents an operational world model with authority, evidence, and consequence attached.

It exists so agents can act with continuity without flattening the human/source reality underneath.

That is the North Star: not better memory as a feature, but continuity as an operating substrate.
